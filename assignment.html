3.1 Reflections

Following are some of my reflections and ideas during buiding this website.

1. Start a prototype as early as possible. I started to create the prototype as the assignment was released. In this way I clearly understand the topic that I want to design and it helped me gather more information and make necessary adjustments before I invested too much time and effort.

2. Consider its interactive when design. As I developed my website more comprehensive, I found the original design was too hard to interact, and I thought the design should be more detail and comprehensive, so I have to refactoring it and start again. It took a lot of effort doing that.

3. Always ready to Iterate and Start Over. I think building a website is a journey, and it's okay to realize that my initial design if it isn't working as well as I thought. Sometimes I have to step back and make the necessary changes. It felt really frustrating, but starting again can lead to a better end result.

4. Stay Organized: I've learned the hard way to keep me code readable and well organized. At first the neatness of the code troubled me for a long time since I could not bear the chaos of the comment and code and I always could not find the code that I wanted to check. So I started learning and borrowing from others' code structures and sorting online, and found the code style that suits me and added clear annotations, which made me very satisfied with the completion of this project.
